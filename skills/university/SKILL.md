# /university — 领域学习路线构建

## 目标 / 边界 / 产出

**目标**：把一个领域变成可执行的**分阶段学习路线**。检索的目的不是挑几本书，
而是建立一张可溯源的「该学什么 → 该读什么 → 按什么顺序 → 现在去找什么」映射。

**边界**（明确不做）：
- **不下钻到书内章节/部分**——书↔课映射到整本书为止；章节级精读规划不在本 skill 范围
- **不复刻培养方案**——实验课/统计/科研训练等不可读化或读者已具备的单元不设课
  （在"读者背景"节用白话解释一句）
- **不做前沿文献深研**——研究生级课程与文献追踪不在本 skill 范围
- **方法论不端上桌**——检索的严谨性留在过程里（本文件 + references 协议），
  产出只给结论和来源链接，不解释分级体系和检索方法

**产出**：**两个文件**，`domains/<slug>/` 下：

- `00-domain.html`（读者向 HTML 档案）：内联 HTML 概览流程图 + CSS 树状路线图 + 当前阶段找书 + 选书权衡
- `system.md`（Markdown，体系详情）：每门课完整书单、来源证据、公开课推荐；低频查阅

硬验收标准：**路线图 5 秒看清每个阶段读哪些书**。

## 步骤 0：确认领域

- `$ARGUMENTS` 有领域名则直接用；否则问用户："要系统学习哪个领域？"
- 定 slug（小写英文，连字符），如 神经科学 → `neuroscience`
- 若 `domains/<slug>/00-domain.html` 已存在：读取并播报进度，问用户是继续、补书单还是重做体系

## 步骤 1：解析激活指令（提示词优先，模糊才补问）

读者激活 skill 时给出的**提示词是第一信息源**，从中提取三个字段：

| 字段 | 例 |
|---|---|
| 领域口径 | "神经科学，打底后往认知走" → 口径已明 |
| 学习目标 | "想彻底搞懂脑科学" → 通识理解 |
| 精读规模 | "先精读一两本" → 已明 |

- 三个字段都提取到 → **零提问**，直接进入检索
- 提取不到或有歧义的字段 → 用 AskUserQuestion 做**一次简要访谈**只补缺口
  （口径歧义最常见也最致命——"神经科学/认知科学/脑科学"选书差异巨大，
  歧义时每个选项注明代表教材和路线含义）
- 背景水平、硬核功底**这里不问**——那要等检索揭示该领域需要什么之后（步骤 3）

## 步骤 2：检索建课程骨架（内部工作，先定"该学什么"，不出书名）

检索方法与来源等级见 [references/retrieval-protocol.md](references/retrieval-protocol.md) 和
[references/source-registry.md](references/source-registry.md)。要点：

1. **学会课程体系**（T1）定主题覆盖：查该领域专业学会的本科课程指南/核心能力框架
   （CS→ACM CS2023；数学→MAA CUPM；统计→ASA GAISE；心理→APA Guidelines；
   生物→AAAS Vision and Change；神经→SfN/FUN core competencies……）
2. **2-3 所 top 校本科培养方案**（T2）定课程单元：fetch degree requirements，提取核心必修课；
   **仅查英语授课院校**（US/UK/AU），中文培养方案不作 T2（中英教材体系独立，混用污染双信号）
3. 两者交叉 → **8-12 门核心课**，每门记录：课名、主题一句话、证据（文件名+链接，供织进叙述）
4. 找不到学会课程体系的领域（哲学、政治学等）：退而用 3-5 所 top 校培养方案交叉
5. 研究生级课程不进骨架（不在本 skill 范围）
6. **课程分型**：顺序课（进路线图，按阅读顺序连续编号）/ 工具课（参考型，不占节点，
   书进阶段书表标"全程伴随查阅"）/ 垫层课（降为一行补底说明，不设课）
7. **提取硬核前置清单**：检索过程中记录该领域绕不开的硬核知识
   （数学/编程/实验/相邻学科——神经科学→生物化学基础、微积分线代；
   CS→离散数学；经济→微积分统计），供步骤 3 访谈用

## 步骤 3：硬核前置访谈（检索后才问得出来的问题）

拿着步骤 2 的硬核前置清单问读者（AskUserQuestion，multiSelect）：
**"这个领域的体系里有这些硬核前置，你具备哪些？"**——每项注明它影响哪些课。

选书规则（步骤 4 执行时引用）：

| 访谈结果 | 选书处理 |
|---|---|
| 具备 | 该课正常配书 |
| 不具备 | 该课**优先选不预设该知识的教材**（如零生物基础 → Bear 而非 Purves）；预设硬核知识的书降为"具备后再深入"选项，一句话里注明门槛 |
| 整门课绕不开该知识 | 课移入自选方向区，注明"需要X基础，具备后再来"；如有合适垫层书给一行补底说明 |
| 已具备超出需要（如统计行家） | 相应课程不设，在"读者背景"解释 |

访谈结果**必须**写进产出的"读者背景"节：白话解释每个设定如何影响了选书。

## 步骤 4：每课书单（内部工作，双信号检索，每课 ≥2 本）

逐课执行，方法细节见 [references/retrieval-protocol.md](references/retrieval-protocol.md)：

1. **双信号交叉**：T2 教学采用 × T3 规范书单；两信号一致 → 核心；单信号 → 候选；
   只有 T4 社区提及 → 不入单
2. **每课 ≥2 本**：1 本主推 + ≥1 本替代，**梯度按步骤 3 访谈结果适配**
3. **一书多课**：全谱系教材按整本书映射（路线图和书表的"角色"体现），不写章节
4. **深浅分级**：路线近端课程深检索（WebFetch 真实 syllabus 确认）；远端浅检索，
   产出白话标"（采用情况待核实）"，读到该课时升级
5. **版次解析**（T5）：主推书核对最新版次；**必查中译本**（书表要填中文书名列）

⚠️ T1-T5、双信号、深浅检索等是**工作语言**，禁止出现在产出文件里。

## 步骤 5：分阶段学习路线

把顺序课分成 **3-4 个阶段**（典型：打地基 → 机制纵深 → 衔接目标方向 → 自选方向）：

- 课程按阅读顺序连续编号；工具课/垫层课不编号
- 最后一个阶段定位"读完主干后自己选方向"，**不用"可选"标签**
- 阶段内选主读书参考"经典组合模式"（见 retrieval-protocol.md），避免两本同级同定位
- 落选的重量级书放入书表替代位，注明何时回来用

## 语言纪律（产出文件强制执行）

- **学校用中文名**：麻省理工 / 约翰霍普金斯 / 加州大学圣迭戈分校 / 斯坦福 / 哈佛 /
  洛桑联邦理工……证据链接照留
- **书名中英双列**：中文书名（认知用）+ 英文书名（搜索/找 PDF 用）；
  无中译本的中文列写释义名并括注"暂无中译"
- **禁词清单**：T1/T2/T3/T4/T5、双信号、深检索/浅检索、蓝图、Tier、主干书带、
  画像豁免、骨架证据、"可选"标签、FUN/SfN 等学会缩写（要提就说全名白话）
- **一句话制**：分阶段路线里每本书 = 一个角色列表项（角色：《书》（作者·版次）——优点一句；
  谁在用＋链接），不写逐本段落——**唯一例外**：『现在做什么』里当前阶段的书（三字段编号列表）
- **段落不手动折行**：一段 = 源文件一行（部分渲染器把单换行画成 `<br>`，中途断行显得怪异）
- **不确定就白话标注**："（采用情况待核实）"、"（版次待核实）"

## 步骤 6：产出两个文件

`mkdir -p domains/<slug>` 后依次生成：

### 文件一：`domains/<slug>/00-domain.html`（读者向 HTML 档案）

**先 `Read references/template.html`**——这是一份完整的范例档案（neuroscience）。**完整复用它的 `<style>` 和底部 `<script>`（交互逻辑），所有 class 名与 JS 一律不改**，只替换文字、书目、封面、锚点 id。

版块（从上到下）与替换规则：

1. **promo 顶条**：`Eatbook` 链接作者 GitHub。
2. **第一屏 hero**：大标题 `<em>领域名</em> 系统学习路线`；`.hero__goal` =「目标：」+ **回显用户目标原话**（不要自加"打底后往 X 走"之类）；`.hero__meta` = `<日期> 为 <用户名> 创建 · N 门课 · N 阶段`；`.routemap` **横向节点路线图**——4 个 `.rm-stage`（`.rm-stage--current` 标当前），每阶段顶 `a.rm-stage-top[href="#phase-N"]` 可跳，每课 `.rm-course` 列该课**全部书**（每本 `a.rm-book[href="#book-<slug>"]`，`.rmb-dot` 用 main/cont/alt/ref 配色区分 主读/续读/备选/参考）。
3. **第二屏 `.vtwrap` 竖向时间轴**（书目主体）：每阶段一个 `.vt-phase[id="phase-N"]`（框线圈起，当前加 `.is-active`）；每课一个 `.vt-course`，**主选书左右交替**（隔课 `--ml`/`--mr`）：主卡 `.vt-main>.vt-card[id="book-<slug>"]`（封面 + 中英书名 + 作者出版社 + 一句简介 + 谁在用）；对面 `.vt-aside` = `.vt-roletag`（课号+角色+一句说明，与主卡相对）+ `.vt-alt-wrap`（备选 `.vt-subcard[id="book-<slug>"]`，更小、虚线连回中轴）。当前阶段的课加 `.is-current`。续读型的课（同一本书跨课）：主卡放该课深入/参考书，`.vt-roletag` 写「与《主线书》对应章节共读」。
4. **进度表 `.progress-wrap`** + **选书权衡 `.rationale-grid`** + **末屏**（skill 简介 + 找书入口：教材 Z-Library / 文献 Sci-Hub / 公开课 MIT OCW·edX）。

**取封面**：每本书 `curl "https://openlibrary.org/search.json?q=<书名+作者>&fields=cover_i&limit=5"`，取第一个有 `cover_i` 的，封面 URL = `https://covers.openlibrary.org/b/id/<cover_i>-L.jpg`，放进 `.cover>img`（带 `onerror="this.remove()"`，取不到时露出 `.cover__fallback` 书名色块）。

**锚点 id 约定**：阶段 `phase-1..N`，书 `book-<英文slug>`；第一屏每个 `a.rm-book[href]` 必须对应第二屏书卡的 `id`，点击才跳得过去。当前阶段在路线图与时间轴都标当前（`--current` / `is-current`）。

### 文件二：`domains/<slug>/system.md`（体系详情，低频查阅）

```markdown
# <领域名> — 分阶段体系详情

> 本文件是 [00-domain.html](00-domain.html) 的配套详情，收录每门课的完整书单、来源说明与公开课推荐。

## 阶段一 · <名>
**目标**：一句话。

**课1 <课名>**（一句话主题）——补充一句：为什么在这个位置/"谁把它当必修"（链接织入）。

- 主读：《中文名》（作者全名 · 版次 年份）——优点一句；谁在用（[链接]）
- 替代/深入/参考：…（与主读的差异一句话）
- 补底/复用/轻松读物/公开课：各一行，按需

（当前阶段的书在 00-domain.md 第三节已有逐本介绍 → 本节只放角色清单 + 链接，不重复）

## 阶段二…四（同构；继续读主读书的阶段在**目标**行写明"《X》全覆盖，无需新书"）
```

**system.md 路线图**：用代码块文本树（阶段 → 课 → 书），不依赖渲染器；主线书跨课标"（续）"，工具书不挂课下、在备注里说明全程随查。

**合同（不可改）**：标题行含领域名（下游读它建术语表）；
进度表列名固定 `| 书 | 入库 | 翻译 | 深读打底 | 共读进度 |`。

## 步骤 7：交付

向用户汇报（表格形式）：
- 骨架：几门顺序课、几个阶段、硬核前置访谈结果如何影响了选书
- 书单：核心/候选各几本、待核实比例（汇报里可以讲方法，文件里不讲）
- **当前阶段找书清单**（逐本，中英文名）
- 下一步：按「现在做什么」清单去获取当前阶段的书，开始阅读
- 会话自然收尾处：≤2 条共创建议
