# 检索协议 — 课程骨架与书单怎么查

**原则：每级来源只能定它有资格定的结论。** 来源分级见 [source-registry.md](source-registry.md)。
课程大纲是一手证据，精选书单是规范性意见，榜单博文是三手噪音。
一本书的"公认"地位由**双信号交叉**判定，不由任何单一来源判定。

## 四步流程

### 第 1 步：建骨架（T1 × T2，不出书名）

1. 搜该领域学会课程指南：
   - `<society> undergraduate curriculum guidelines <领域>`
   - `<领域> core competencies undergraduate`
2. 搜 4-5 所 top 校培养方案：
   - `<学校> <领域> major requirements core courses`
   - `<学校> <领域> undergraduate degree course map`
3. 交叉规则：学会框架定主题覆盖完整性，培养方案定课程切分单元。
   两边都出现的主题 → 必选课；只在培养方案出现 → 看 2-3 校是否一致；
   只在学会文件出现且无对应课 → 并入相邻课程做主题备注。
4. 产出 8-12 门，每门记录骨架证据（文件名+链接）。

### 第 2 步：双信号书单（逐课）

**描述性信号（T2）——"谁在教学中真的用它"：**
- `MIT OCW <领域/课名>`、`site:ocw.mit.edu <课名>`
- `<学校> <课号> syllabus required textbook`
- `<学校> <领域> <课名> syllabus filetype:pdf`
- 牛剑：`<faculty> guide to courses reading list`（剑桥 Tripos 类指南常带书名+出版社）
- **深检索必须 WebFetch 真实 syllabus 页面/PDF 确认 required/recommended 字段，不信搜索摘要**

**规范性信号（T3）——"专家认为该按什么顺序读什么"：**
- 查该领域的成名精选书单（见 source-registry.md T3 清单）
- `<领域> reading list site:github.com`（OSSU/awesome-* 类）
- Stack Exchange：`<领域> book recommendation big-list site:*.stackexchange.com`
- 记录精选书单给出的：顺序、梯度（本科→研究生）、版本对比意见

### 第 3 步：交叉定级

| 信号组合 | 定级 | 处理 |
|---|---|---|
| T2 采用 ∧ T3 推荐 | **核心** | 入书单，可做主推 |
| 仅 T2（≥2 校采用） | 核心 | 入书单（教学采用是最强单信号） |
| 仅 T2（1 校） | 候选 | 入书单做替代位，注明单源 |
| 仅 T3 | 候选 | 入书单做替代位（精选书单有编辑偏好） |
| 仅 T4（社区提及） | 不入单 | 最多在备注提一句 |

每课 **≥2 本**：1 本主推 + ≥1 本替代。替代必须与主推有真实差异
（视角：生物的/计算的/临床的；难度：导论/重型；体系：美/英教材传统），不收"同款备份"。

### 第 4 步：版次解析（T5）

- `<书名> latest edition <出版社>`，主推书必须核对到最新版次+出版年月
- 版次敏感领域（生物/医学/CS 应用层）必须新版；数学/物理经典老版常无妨
- 中译本顺手查（书名+"中译本/中文版"），有则在书单注明

## 深浅检索分级（控制成本）

| 级别 | 适用 | 要求 |
|---|---|---|
| **深** | 精读路线近端课程（前 2-3 门） | 完整四步；T2 证据必须 WebFetch 原始 syllabus 确认；版次核对到年月 |
| **浅** | 远端课程 | T3 精选书单 + 模型知识打底即可；每个未经 T2 确认的条目标 `[待核实]`；读到该课时升级为深检索 |

浅检索不是偷工：骨架先有完整地图比每个节点都精确更重要（最小摩擦——
地图长期是空的会让"下一步读什么"每次都变成新项目）。但浅条目**必须可识别**，
不得与深证据条目混排冒充（同防幻觉铁律的 `[模型补充]` 精神一致）。

## 证据落盘格式

书单中每本书一行证据，缺一项就补查或标注：

```
谁在用：<学校> <课号>（<学期/年>，required）— <链接>；书单：<OSSU/Rigetti/…> — <链接>
```

浅检索条目：

```
书单：<来源> — <链接>；教学采用 [待核实]
```

## 评分维度（每课定主推时用）

| 维度 | 问题 |
|---|---|
| 权威采用度 | 几所 top 校用？required 还是 recommended？ |
| 主干覆盖 | 单本能覆盖该课多大比例的核心议题？ |
| 教学性 | 写给本科生还是研究生？图表质量？有无习题？ |
| 互补性 | 和同课替代书的视角/层次是否真有差异？ |
| 梯度匹配 | 是否匹配用户访谈出的背景水平？ |

## 经典组合模式（精读路线用）

- **导论 + 主干**：1 本友好导论（先通读）+ 1 本重型标准教科书（精读主力）
- **主干 + 特色**：标准教科书 + 1 本特别视角（计算的/历史的/批判的）
- 避免选 2 本同级别同定位的书做精读（如神经科学同时精读 Kandel 和 Purves——
  二选一，另一本留在书单替代位做参考书）

## 常见陷阱

- 榜单博文互相抄袭，全网同一份名单 ≠ 公认（用双信号规则过滤）
- 把研究生教材当本科核心（看课号层级；研究生书进 branch 不进骨架书单）
- 忽略出版年：对比类长文常年不更新——本次实例：Neuroamer 2016 说 Berkeley 用 Kandel，
  实际 MCB 160 现行 required 已换 Luo《Principles of Neurobiology》。**采用情况以一手 syllabus 为准**
- 美/英体系差异：同一领域两边经典不同，按用户目标选，书单可并收并注明体系
- 科普书混入教材榜单（科普 ≠ 教科书；科普可列入"配套读物"不占 ≥2 本名额）
